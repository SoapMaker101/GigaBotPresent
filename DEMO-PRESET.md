# Демо-пресет для презентации CSM

Краткая инструкция по подготовке окружения для стабильного демо по сценарию из [docs/06-Scenarij-demo-CSM.md](docs/06-Scenarij-demo-CSM.md).

## Режим презентации (рекомендуется для демо)

Чтобы на демо были доступны только нужные инструменты (без exec, web, spawn, ocr, генерации изображений, голосовых заметок), включите режим презентации:

- **Через переменную окружения:** `GIGABOT_AGENT__PRESENTATION_MODE=true` перед запуском `gigabot gateway` или `gigabot agent`.
- **Через конфиг:** в `~/.gigabot/config.json` в секции `agent` добавьте `"presentationMode": true`.

В режиме презентации доступны: file, project, message, cron, knowledge, tasks, summary.

## Что нужно к началу демо

1. **Рабочий бот** — конфиг GigaChat (и при необходимости Telegram) настроен, бот запускается (`gigabot gateway` или `gigabot agent`).
2. **Один проект** — например «Презентация CSM» с папкой «Документация».
3. **1–2 файла в Документация** — PDF или DOCX с текстом (можно про метрики, внедрение, регламенты), чтобы показать индексацию и поиск по RAG.
4. **3–5 задач** — с разными приоритетами (high/medium/low) и дедлайнами (сегодня, завтра, через неделю), чтобы показать список и отчёт по приоритизации.

## Шаги подготовки (через бота)

Выполни в Telegram (или в интерактивном агенте) по порядку:

| Шаг | Команда / запрос |
|-----|-------------------|
| 1 | Создай проект «Презентация CSM». |
| 2 | (Положи вручную в папку Документация проекта 1–2 файла или отправь файлы боту и попроси положить в проект в Документацию.) |
| 3 | Создай базу знаний для проекта «Презентация CSM» и добавь в неё папку Документация. |
| 4 | Добавь задачу: подготовить слайды до завтра 18:00, приоритет высокий. |
| 5 | Добавь задачу: согласовать смету с клиентом, приоритет средний. |
| 6 | Добавь задачу: отправить еженедельный отчёт, дедлайн через 5 дней, приоритет низкий. |

## Проверка перед презентацией

- [ ] «Найди в базе знаний всё про [тема из документа]» — бот возвращает фрагменты из документов.
- [ ] «Покажи мои задачи» — список с приоритетами и дедлайнами.
- [ ] «Подготовь отчёт по задачам: что сделать первым по критичности и дедлайну» — текстовый отчёт с упорядоченным списком.
- [ ] «Дай сводку: что загружено, какие задачи, что в базах знаний» или «Вызови summary» — блоки «Проекты», «Задачи», «Базы знаний RAG».

После этого можно проводить демо по таймингу из [docs/06-Scenarij-demo-CSM.md](docs/06-Scenarij-demo-CSM.md).
