# Навык: отчёт по приоритизации задач

Когда пользователь просит подготовить отчёт по задачам (что сделать первым, что позже, по критичности / сложности / дедлайну / приоритету):

1. Вызови **tasks(action="list")** без фильтров (или с фильтром по project, если пользователь указал проект).
2. В ответе будут задачи с полями: id, title, project, status, priority (low/medium/high), deadline (ISO), created_at.
3. Сформируй текстовый отчёт:
   - Сначала перечисли задачи с **priority="high"** и с ближайшим **deadline** (срочные и важные).
   - Затем задачи **medium** и с дедлайном в ближайшие дни.
   - В конце — **low** и без дедлайна или с отдалённым дедлайном.
   - Для каждой задачи укажи: название, приоритет, дедлайн (если есть), проект (если есть).
4. Добавь краткую рекомендацию: «Рекомендуется выполнить в первую очередь: …» на основе приоритета и срочности.

Приоритет в системе (priority) трактуй как **критичность**; **сложность** при отсутствии отдельного поля можно не выводить или указать «не задана».
