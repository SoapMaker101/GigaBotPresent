# GigaBot — форк для презентации CSM в Сбере

Это форк GigaBot, настроенный под сценарий презентации для CSM: внедрение LLM в контур, демо RAG и задач, метрики, переход к GigaEnterprise.

## Отличия от базового GigaBot

- **Tool `summary`** — сводка по контуру: проекты (папки), задачи (статистика), базы знаний RAG. Запросы вида «что загружено», «какие задачи», «аналитика» обрабатываются через этот инструмент.
- **Навык `task_report`** — отчёт по приоритизации задач (по приоритету и дедлайну). Запрос «подготовь отчёт по задачам: что сделать первым по критичности и дедлайну» выполняется через tasks(list) и этот навык.

## Подготовка демо (пресет)

1. Запусти бота и выполни onboarding: `gigabot onboard` (или используй существующий `~/.gigabot`).
2. Создай проект для демо: в Telegram или CLI напиши «Создай проект „Презентация CSM“.»
3. Положи в папку `workspace/projects/Презентация CSM/Документация/` 1–2 файла (PDF или DOCX) с текстом про метрики или внедрение.
4. Попроси бота: «Создай базу знаний для проекта „Презентация CSM“ и добавь в неё папку Документация.»
5. Добавь 3–5 задач с разными приоритетами и дедлайнами, например:
   - «Добавь задачу: подготовить слайды, дедлайн завтра 18:00, приоритет высокий»
   - «Добавь задачу: согласовать смету, приоритет средний»
   - «Добавь задачу: отправить отчёт, дедлайн через 3 дня, приоритет низкий»
6. Проверь сценарии из [../docs/06-Scenarij-demo-CSM.md](../docs/06-Scenarij-demo-CSM.md): база знаний + поиск, список задач, запрос отчёта по приоритету/дедлайну, сводка (summary).

Установка и конфиг — как в базовом GigaBot (см. корневой README репозитория или pyproject.toml).
